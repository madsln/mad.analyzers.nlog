Exception should be passed as the first argument to the log method

When logging an exception, the Exception object must be passed as the first argument to the NLog logging method. NLog recognises the first argument specially: if it is an Exception, NLog captures the full exception details, message, and stack trace in dedicated log-event properties.

Passing the exception as any other argument causes it to be serialised as a plain string via ToString() and the rich exception metadata (stack trace, inner exceptions) is lost.

Non-compliant code

catch (Exception ex)
{
    logger.Error("Order {OrderId} processing failed: {Error}", orderId, ex);
}

Compliant code

catch (Exception ex)
{
    logger.Error(ex, "Order {OrderId} processing failed", orderId);
}
