Log message template should use PascalCase for named placeholders

A named placeholder used in an NLog message template should follow PascalCase, a naming convention where the first letter of each compound word is capitalised (e.g. {UserId} instead of {userId} or {user_id}).

This naming convention is strongly recommended for structured logging because each named placeholder becomes a property name in the structured log event. Consistent PascalCase property names improve searchability and query consistency in log aggregation tools such as Seq, Elastic, or Loki.

Non-compliant code

logger.Info("User {userId} logged in from {ipAddress}", userId, ip);

Compliant code

logger.Info("User {UserId} logged in from {IpAddress}", userId, ip);
