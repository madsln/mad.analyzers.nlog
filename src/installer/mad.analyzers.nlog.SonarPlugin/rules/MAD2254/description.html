Log message template should be a static (compile-time constant) expression

The message template passed to an NLog logging method should be a static string literal or a const expression. Building a template dynamically at runtime (e.g. using string concatenation or interpolation) destroys the structured metadata — the placeholder names are lost and log aggregation tools can no longer index individual properties consistently.

Dynamic templates also prevent static analysis (including this analyser) from validating placeholder counts and names at compile time.

Non-compliant code

string template = $"Processing order {{OrderId}} for {customerType} customer";
logger.Info(template, orderId);

Compliant code

logger.Info("Processing order {OrderId} for {CustomerType} customer", orderId, customerType);
