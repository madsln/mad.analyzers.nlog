Named placeholders in log message templates should not be purely numeric

Named placeholders in NLog message templates should have descriptive, non-numeric names. A placeholder such as {0} or {1} looks like a string.Format index and does not produce a meaningful property name in the structured log event, undermining the value of structured logging.

Non-compliant code

logger.Info("Processing order {0} for customer {1}", orderId, customerId);

Compliant code

logger.Info("Processing order {OrderId} for customer {CustomerId}", orderId, customerId);
