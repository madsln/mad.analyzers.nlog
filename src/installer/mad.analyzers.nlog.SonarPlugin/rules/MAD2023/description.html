Invalid braces in log message template

NLog message templates use curly braces { and } to denote named placeholders for structured values. Unbalanced or incorrectly nested braces are not valid placeholder syntax and can cause runtime exceptions or completely unexpected logging behaviour.

Common mistakes include:

  • Using a single opening brace without a closing brace: "Value is {Value"
  • Using a single closing brace without an opening brace: "Value is Value}"
  • Mixing format strings from string.Format with NLog template syntax

Non-compliant code

logger.Info("Processing {OrderId} failed}", orderId);   // extra closing brace
logger.Info("Processing {OrderId failed", orderId);    // missing closing brace

Compliant code

logger.Info("Processing {OrderId} failed", orderId);
