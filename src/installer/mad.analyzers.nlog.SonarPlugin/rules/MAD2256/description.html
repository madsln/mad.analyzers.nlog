Avoid inline serialization of objects in log message arguments

Calling JsonSerializer.Serialize(), JsonConvert.SerializeObject(), or .ToString() on a complex object before passing it to an NLog logging method produces a flat string and bypasses NLog's structured capture pipeline.

Instead, pass the original object and use the {@Property} destructuring prefix to let NLog capture the object's structure, or the {$Property} prefix to force explicit stringification via NLog's own serialisation pipeline. For consistent behaviour across all call sites, register a custom IValueFormatter with NLog's configuration.

Serialising inline also incurs the full serialisation cost even when the configured log level is disabled, which can impact performance in hot paths.

Non-compliant code

logger.Info("Order received: {Order}", JsonSerializer.Serialize(order));
logger.Info("Customer: {Customer}", customer.ToString());

Compliant code

// Let NLog destructure the object (preserves structure in structured sinks)
logger.Info("Order received: {@Order}", order);

// Force NLog stringification via its own pipeline
logger.Info("Order received: {$Order}", order);
