## Architektur-Entscheidung

### ADR-008: LineEnding-Metadatum ersetzt IsStringLiteralStart-Heuristik

**Problem:** ADR-007 löste abweichende `startOffset`-Werte zwischen manuell (CRLF) und
generiert (LF) geschriebenen Demo-Dateien durch eine Dual-Formel mit Inhalts-Heuristik
(`IsStringLiteralStart`). Diese Heuristik ist zufällig korrekt, nicht strukturell garantiert,
und würde bei einer dritten Konvention oder einer Nachrichtenvorlage, die mit `"`, `$` oder `@`
beginnt, versagen.

**Entscheidung:** Der Zeilenenden-Typ wird einmalig beim Einlesen der Quelldatei erkannt
(vor jeder Normalisierung), als `LineEndingKind` durch die Pipeline weitergereicht, und
`ResolveAbsolutePosition` wählt die Formel ausschließlich anhand dieses Metadatums.

→ Vollständig dokumentiert in [ADR-008](../adr/ADR-008-LineEnding-Metadata.md).

## Implementierungsdetails

### `LineEndingKind`-Enum (Models.cs)

```csharp
internal enum LineEndingKind { LF, CRLF, Mixed }
```

### `LineEndingDetector.Detect`

```csharp
bool hasCRLF = rawContent.Contains("\r\n");
bool hasLF   = rawContent.Replace("\r\n", "").Contains('\n');

return (hasCRLF, hasLF) switch
{
    (true,  true)  => LineEndingKind.Mixed,
    (true,  false) => LineEndingKind.CRLF,
    _              => LineEndingKind.LF,
};
```

Erkennung erfolgt auf dem unberührten `rawContent`, bevor `SourceCleaner` Zeilenenden
normalisiert.

### `DiagnosticMarkerInserter.ParseLineEndingOverride`

Liest das optionale Header-Feld `// line-ending: crlf` aus dem Kommentar-Header.
Erlaubte Werte: `lf`, `crlf`, `mixed` (case-insensitive). Fehlt das Feld → `null`.

### Neue Formel in `ResolveAbsolutePosition`

```csharp
int absPos = lineEnding == LineEndingKind.CRLF
    ? classStart + (startOffset - crlfCount)      // CRLF-Quelle: manuelle Konvention
    : classStart + (startOffset - crlfCount + 2); // LF-Quelle:   ADR-004-Formel
```

`IsStringLiteralStart` und die gesamte Dual-Kandidaten-Logik sind entfernt.

### Pipeline-Änderung in `TestCompiler.ProcessFile`

```
rawContent einlesen
  → LineEndingDetector.Detect(rawContent)          // ADR-008: vor jeder Normalisierung
  → SourceCleaner.SplitHeaderAndBody(rawContent)
  → DiagnosticMarkerInserter.ParseLineEndingOverride(header)  // optionales Feld hat Vorrang
  → DiagnosticMarkerInserter.InsertMarkers(body, diagnostics, lineEnding)
  → ...
```

## Rückwärtskompatibilität

Bestehende Demo-Dateien benötigen **keine Anpassung**:
- Manuell geschriebene Dateien haben CRLF-Zeilenenden → werden automatisch als
  `LineEndingKind.CRLF` erkannt.
- Generierte Dateien haben LF-Zeilenenden → werden als `LineEndingKind.LF` erkannt.
- Das optionale Header-Feld `line-ending:` ist in keiner bestehenden Datei gesetzt und
  wird daher ignoriert.
