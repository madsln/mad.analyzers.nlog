## Inhalt im Detail

### `Models.cs`

Alle Records liegen im Namespace `TestDataGen` (kein Unter-Namespace – → ADR-003).

| Record | Zweck |
|---|---|
| `DemoVariant` | Vollständig aufgelöste Demo-Klassenvariante, die `CodeFileBuilder` und `MetadataHeaderBuilder` als Eingabe erhalten |
| `DiagnosticDescriptor` | Ein einzelner Diagnostic-Eintrag aus dem Demo-Header (`// MAD2017=offset:len;int=n;int=m`), mit typisierter `Parameters`-Liste |
| `TestMethodSpec` | Eine generierte Testmethode (prozessierter Code + Diagnostics) |
| `TestClassSpec` | Eine generierte Testklasse (Klassenname, Ausgabepfad, Methoden-Liste) |

Alle Records sind `internal`.  
`DiagnosticDescriptor.Parameters` ist `IReadOnlyList<(string Type, string Value)>` – ein Value-Tuple-Ansatz ohne extra Record (→ ADR-003).

### `IO/OutputWriter.cs`

- `Write(string filePath, string content)` erstellt fehlende übergeordnete Verzeichnisse via `Directory.CreateDirectory` und schreibt via `File.WriteAllText` mit `UTF8Encoding(encoderShouldEmitUTF8Identifier: false)` (kein BOM).
- Klasse ist `internal`, nicht statisch (wird per Konstruktor-Injektion an `DemoGenerator` und `TestCompiler` übergeben – Vorbereitung für WL-04/WL-05).
