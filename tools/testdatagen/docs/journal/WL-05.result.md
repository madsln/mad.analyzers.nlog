## Architektur-Entscheidung

### ADR-005: DemoGenerator-Ausgabestruktur

**Problem:** Der `DemoGenerator` schrieb Dateien nach `{outDir}/{loggertype}/`, der
`TestCompiler` erwartet aber `{source}/singlediag/`, `{source}/nodiag/`, `{source}/usings/`.

**Entscheidung:** `DemoGenerator` schreibt ab sofort nach
`{outDir}/singlediag/MAD2017/{loggertype}/`. Dadurch ist `demo --out ./tmp/demo` direkt
als `--source ./tmp/demo` für `test` verwendbar.

→ Vollständig dokumentiert in [ADR-005](../adr/ADR-005-DemoGenerator-Output-Structure.md).

## Inhalt im Detail

### `SourceCleaner`

- `Clean(source)`: entfernt alle `//'`-Zeilen und Leerzeilen (Whitespace-only), normalisiert
  Zeilenenden zu `\n`.
- `SplitHeaderAndBody(source)`: trennt führende `//`-Zeilen (Metadaten-Header) vom Body.

---

### `LogLevelParameterizer`

- Ersetzt den **ersten** gefundenen Log-Level-Bezeichner (`Trace|Debug|Info|Warn|Error|Fatal`)
  durch `%LOGLEVEL%` (Regex mit Word-Boundary, Case-Sensitive).
- Da alle generierten Demo-Dateien `Info` verwenden, wird stets `Info` ersetzt.

---

### `PreambleResolver`

- Durchsucht den Quelltext nach Schlüsselwörtern aus der Usings-Tabelle (Word-Boundary-Regex).
- Gefundene Preamble-Blöcke werden durch `\n\n` getrennt dem Code vorangestellt.
- Für MAD2017-Dateien sind keine Preamble-Einträge aktiv (Usings-Verzeichnis leer).

---

### `DiagnosticMarkerInserter`

**Header-Parser:**  
Parst `// MAD2017=330:29;int=1;int=2` → `DiagnosticDescriptor(Id="MAD2017", StartIndex=330, Length=29, Parameters=[("int","1"),("int","2")])`.

**Marker-Einfügung – Offset-Invertierung (Kernproblem):**  
Der im Header gespeicherte `StartOffset` wurde vom `MetadataHeaderBuilder` (WL-04) nach
der Formel `startOffset = (msgPos - classPos) + crlfCount - 2` berechnet (ADR-004). Dieser
Wert ist **kein** direkter Byte-Offset in den Raw-Body.

Um die tatsächliche Zeichenposition zu rekonstruieren, invertiert `ResolveAbsolutePosition`
die Formel:

```
absPos = classPos + startOffset - crlfCount + 2
```

Dabei wird `crlfCount` aus dem Raw-Body (Fenster von `classPos+1` bis `classPos+startOffset+slack`)
berechnet. Die Methode konvergiert in einem Schritt, da das Fenster alle relevanten `\n`-Zeichen
enthält.

Marker werden in **absteigender** StartIndex-Reihenfolge eingefügt, damit frühere Offsets
durch die Einfügung nicht verfälscht werden.

**Wichtig:** Marker werden **vor** `SourceCleaner.Clean()` eingefügt, damit die Offset-Berechnung
auf dem ungeinigten Body (inklusive Leerzeilen) korrekt ist. Beim nachfolgenden `Clean()` werden
leere Zeilen entfernt – der Marker auf der Logging-Aufruf-Zeile bleibt erhalten.

---

### `CodeSampleWrapper`

Repliziert `IndentTextBlock -indent_spaces 4` + `CreateCodeSample` aus `CompileTestData.ps1`:

```csharp
using System;
using NLog;

namespace analyzer.test
{
    {4-space-indentierter Code}
}
```

---

### `VerbatimEscaper`

`"` → `""` für C# Verbatim-Strings.

---

### `TestMethodBuilder`

**nodiag:** `[DataTestMethod]` + 6 `[DataRow]` + `await VerifyCS.VerifyAnalyzerAsync(test)`  
**singlediag:** `[TestMethod]` + 6 `[DataRow]` + `expected0`-Variable + `WithArguments` + `await VerifyCS.VerifyAnalyzerAsync(test, expected0)`

Parameter-Formatierung:
- `int` → Integer-Literal (z. B. `1`)
- Sonstige → String-Literal (z. B. `"foo"`)

---

### `TestClassBundler`

- `ToClassName(relativePath)`: ersetzt `\` und `/` durch `_`, hängt `_Tests` an.
  Beispiel: `singlediag/MAD2017/logger` → `singlediag_MAD2017_logger_Tests`.
- `Render(spec, isSingleDiag)`: erzeugt vollständige `.cs`-Datei mit 8-Space-Einzug
  für Methoden im Klassen-Body.

---

### `TestCompiler`

Orchestrierung:
1. Usings-Tabelle laden aus `{source}/usings/` (graceful wenn Verzeichnis fehlt).
2. `singlediag/` und `nodiag/` getrennt verarbeiten (jeweils rekursiv).
3. Dateien gruppieren nach relativem Unterverzeichnis → je ein `TestClassSpec`.
4. Für jede Datei: `SplitHeaderAndBody` → `ParseHeader` → `InsertMarkers` → `Clean` →
   `Parameterize` → `Resolve` → `Wrap` → `Escape` → `TestMethodSpec`.
5. Pro Gruppe: `TestClassBundler.Render()` → `OutputWriter.Write()`.

## Abnahmekriterium – Ergebnis

```
dotnet run -- demo --out ./tmp/demo-new
→ demo: 169 files generated in ./tmp/demo-new
  Struktur: tmp/demo-new/singlediag/MAD2017/{logger,ilogger,iloggerextensions}/

dotnet run -- test --source ./tmp/demo-new --out ./tmp/tests-new
→ test: 3 test-class files written to ./tmp/tests-new
  Dateien:
  - singlediag/singlediag_MAD2017_logger_Tests.cs
  - singlediag/singlediag_MAD2017_ilogger_Tests.cs
  - singlediag/singlediag_MAD2017_iloggerextensions_Tests.cs
```

**Stichproben-Check:**  
`{|#0:""Message with {Placeholder1}""|}`-Marker korrekt um die Message-Template-Strings
positioniert. `WithLocation(0)` und `WithArguments(1, 2)` korrekt generiert.
