## Inhalt im Detail

### `ClassNameBuilder`

Portiert `generate_demo_title()` aus dem Python-Referenzskript 1:1:

```
{LoggerType}_Log_[WithException_]{PascalCaseTestcaseName}[_{ObjectArray|ReadOnlySpan}]
```

`PascalCaseTestcaseName` wird durch `capitalize()` pro Wort gebildet (erstes Zeichen groß,
Rest klein – exakt wie Pythons `str.capitalize()`).

---

### `VariantBuilder`

Erzeugt pro Kombination `LoggerType × CallTemplate × TestCase` bis zu drei `DemoVariant`-Objekte:

| Variante | Suffix | Ausschluss |
|---|---|---|
| Direct | `""` | – |
| ObjectArray | `"ObjectArray"` | – |
| ReadOnlySpan | `"ReadOnlySpan"` | `HasException` **oder** `ILogger` **oder** `HasCultureInfo` |

Argument-Strings:
- **Direct:** `message, arg0, arg1` direkt im Call
- **ObjectArray:** `var objectArray = new object[] { arg0, arg1 };` + Call mit `objectArray`
- **ReadOnlySpan:** `var objectSpan = new ReadOnlySpan<object>( new object[] { arg0, arg1 } );` + Call mit `objectSpan`

Exception-Deklaration (`ArgumentConfig.ExceptionDeclaration`) wird bei `HasException`
als erste Zeile im Method-Body eingefügt.

---

### `MetadataHeaderBuilder`

1:1-Port der Python-Formel (inkl. des undokumentierten `-2`):

```csharp
int startOffset = messageStart - classStart + crlfCount - 2;
```

`crlfCount` = Anzahl `\n` im Substring `fileBodyCode[classStart+1 .. messageEnd]`.  
→ ADR-004 erklärt die Zeilenenden-Strategie.

---

### `CodeFileBuilder`

Repliziert das Python-`template_class_body` mit `\n`-Zeilenenden:

```
{usings}
\n
namespace {ns};
\n
public class {ClassName}
{
    {LoggerDeclaration}
\n
    public void ShouldRaiseDiagnostic_MAD2017()
    {
        {methodBody}
    }
}
```

`Build()` ruft `BuildBody()` → `MetadataHeaderBuilder.Build()` → prepend Header.

---

### `DemoGenerator`

Orchestriert: `LoggerTypes × CallTemplates × TestCases`, ruft `VariantBuilder.Build()`,
dann `CodeFileBuilder.Build()`, dann `OutputWriter.Write(outDir/{loggertype}/{ClassName}.cs)`.
Gibt Anzahl geschriebener Dateien zurück.
